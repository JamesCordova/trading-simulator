Proyectos de Curso – Laboratorio de Calidad de Software 
Equipos: 5 estudiantes 
Duración sugerida: 10 días de trabajo efectivo 
Opción A: Diseño e implementación de un Plan de Pruebas completo 
1. Objetivo 
Diseñar e implementar un plan de pruebas completo para una aplicación de tamaño pequeño o 
mediano, que incluya pruebas unitarias, de integración, de sistema y de aceptación, definiendo 
casos de uso, criterios de entrada/salida y una estrategia básica de automatización que permita 
medir y mejorar la calidad del software. 
2. Base conceptual 
Conceptos clave: 
• - Ciclo de vida de pruebas de software (planificación, diseño, ejecución, cierre). 
• - Tipos de pruebas: unitarias, integración, sistema, aceptación del usuario. 
• - Trazabilidad entre requerimientos, casos de uso y casos de prueba. 
• - Automatización de pruebas (frameworks xUnit, pruebas funcionales y de API). 
• - Métricas de calidad: cobertura, número de defectos, severidad, tasa de fallos.

4. Herramientas de software y base de datos 
Lenguaje/entorno sugerido: Java, Python o similar; cada equipo puede elegir según los cursos 
previos. 
• Herramientas sugeridas: 
• - Framework de pruebas unitarias: JUnit (Java) – sitio oficial: https://junit.org/ 
• - Herramientas de automatización funcional/web: Selenium WebDriver – 
https://www.selenium.dev/ 
• - Pruebas de API REST: Postman – https://www.postman.com/downloads/ 
• - Gestión de código fuente: Git + GitHub / GitLab. 
• - IDEs sugeridos: Visual Studio Code, IntelliJ IDEA Community, Eclipse, PyCharm Community. 
• Base de datos sugerida (una de las siguientes): 
• - PostgreSQL: SGBD relacional de código abierto – https://www.postgresql.org/download/ 
• - MySQL Community Server: https://dev.mysql.com/downloads/ o 
https://www.mysql.com/products/community/ 
5. Resultados esperados 
• - Documento de Plan de Pruebas (incluyendo alcance, estrategia, ambiente, riesgos). 
• - Especificación de casos de prueba (unitarios, integración, sistema, aceptación). 
• - Conjunto de pruebas automatizadas ejecutables. 
• - Reporte de ejecución de pruebas, defectos identificados y métricas básicas. 
• - Presentación final explicando el enfoque, resultados y lecciones aprendidas.
